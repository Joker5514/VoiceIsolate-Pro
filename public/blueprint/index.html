Thread Pool Orchestration

Dynamic thread allocation across CPU/GPU cores. The ThreadManager spawns a configurable pool (default: navigator.hardwareConcurrency) of Web Workers, each assigned to pipeline stages via a priority queue. GPU compute threads handle STFT/iSTFT and ML inference in parallel, while CPU workers manage time-domain passes and I/O.


Single-Pass Spectral Architecture

Critical design principle: one STFT → all spectral operations in-place → one iSTFT. Eliminates cumulative phase smearing and echo/doubling artifacts caused by multiple spectral round-trips. All 12 spectral stages execute on the same complex buffer between transforms.

Modular Node Graph

Every processing stage is a self-contained node implementing the DSPNode interface: process(buffer), configure(params), getMetrics(). Nodes connect via typed channels (Float32, Complex64, SpectralFrame). Plugin API exposes the same interface for custom modules.


Asynchronous Job Queue

BatchOrchestrator manages large-scale processing via a priority queue with configurable concurrency (1–64 simultaneous files). Jobs stream progress via MessageChannel, enabling real-time UI updates. Failed jobs auto-retry with exponential backoff.

04 — Algorithms & Models

Model Selection Rationale

Each model chosen for browser-native execution via ONNX Runtime Web with WebGPU/WASM backends. No server round-trips, no data egress.

Demucs v4.1
Primary Source Separation
The main AI engine that separates vocals from music, background noise, and other sounds. Processes audio at 2.5x real-time speed.
Band-Split RNN
Secondary Separation
Works alongside Demucs to catch details it might miss. Together they achieve better results than either alone.
ECAPA-TDNN
Speaker Identification
Creates a unique "voiceprint" for each speaker. Learns a voice from just 3 seconds of audio to track and isolate specific people.
Silero VAD v5
Voice Activity Detection
Detects when someone is speaking vs silence. Helps the system know where to focus and saves processing power.
HiFi-GAN v2
Voice Reconstruction
Rebuilds natural-sounding voice from processed audio. Makes voices sound human again after heavy noise removal.
Conformer
Final Polish
The finishing touch that removes any remaining digital artifacts and ensures clean, professional output.

07 — Optimization Strategies

Low-Latency Engineering

⚡

GPU-Accelerated STFT

WebGPU compute shaders execute FFT butterfly operations in parallel across frequency bins. Single-pass architecture means only 2 GPU transfers (forward + inverse) per audio chunk instead of 2N for N spectral stages. 8× throughput vs CPU FFT.

⊞

SharedArrayBuffer Ring Buffer

AudioWorklet ↔ Worker communication via lock-free ring buffer on SharedArrayBuffer. Zero-copy data transfer eliminates postMessage serialization overhead. 128-sample chunk size enables <3ms audio thread latency.

⊗

ONNX Runtime WebGPU

ML models quantized to INT8 via ONNX Runtime's WebGPU execution provider. Demucs inference: ~400ms per 10s chunk (vs 1.8s on WASM). Dynamic batching groups sequential chunks for GPU efficiency.

⊘

Adaptive Pipeline Bypass

VAD-driven stage skipping: silent segments bypass all processing. Low-noise segments skip spectral subtraction. Single-speaker audio skips diarization. Reduces average processing time by 40% on typical content.

08 — Security & Privacy

Privacy-First Architecture

100% local processing. Zero telemetry. No audio data ever leaves the device.

🔒

Local-Only Processing

All ML models loaded from local cache or bundled ONNX files. No API calls, no cloud inference, no audio upload. Network requests blocked during processing via Content Security Policy.

🛡

AES-256 Encryption at Rest

Temporary processing buffers encrypted in memory. Exported files optionally encrypted with user-provided key. Cache purged on session end with secure overwrite.

📋

Forensic Chain of Custody

SHA-256 hashes at input, each processing stage output, and final export. Cryptographic audit log cannot be tampered with. Timestamps, parameter snapshots, model versions — all recorded.

🚫

Zero Telemetry

No analytics, no usage tracking, no crash reporting that includes audio data. Optional anonymous feature-usage stats (opt-in only). Open-source audit trail for all network calls.
