- **Privacy-first:** All processing local (no server, no data leakage)

### 2.2 12-Node DSP Pipeline (Detailed Specs)

#### **NODE 1: Input Normalization & Format Detection**
```
Purpose: Load audio/video, detect format, normalize to float32 PCM
Input:  File (MP3, WAV, MP4, MOV, WEBM) or ArrayBuffer
Output: Float32Array @ 44.1√Ę‚ā¨‚Äú48 kHz mono or stereo

Process:
  1. Detect MIME type (audio/mpeg, audio/wav, video/mp4, etc.)
  2. Extract audio track (if video) via ffmpeg.wasm or native codec
  3. Resample to target (44.1 or 48 kHz) using polyphase filter
  4. Convert to float32 PCM (-1.0 to +1.0 range)
  5. Normalize peak amplitude to 0 dB (or user-specified)
  6. Return: { sampleRate, channels, duration, buffer }
```

#### **NODE 2: Noise Profiling (Automatic)**
```
Purpose: Learn noise signature from silent regions
Input:  float32 PCM, sample rate, duration
Output: Noise power spectrum (dB) @ FFT bin resolution

Algorithm:
  1. Extract first 1√Ę‚ā¨‚Äú2 seconds (typically silent)
  2. Compute STFT: FFT size 4096, hop 1024 samples
  3. Frame-by-frame power: P(k) = |X(k)|√ā¬≤ / N√ā¬≤
  4. Average over frames: P_noise(k) = mean(P(k))
  5. Smooth spectrum (moving average, 5-bin kernel)
  6. Apply floor: P_noise = max(P_noise, 1e-10)
  7. Convert to dB: dB = 10*log10(P_noise)
  8. Return: 1D array [FFT_bins] of noise power

Fallback (if silent region unavailable):
  √Ę‚Ä†‚Äô Use statistical median of entire signal's low-energy frames
  √Ę‚Ä†‚Äô Or: User-supplied noise profile (external recording)
```

#### **NODE 3: FFT-Based Spectral Decomposition**
```
Purpose: Transform time-domain audio to frequency domain
Input:  float32 PCM buffer, FFT size (4096√Ę‚ā¨‚Äú16384), hop size
Output: 2D magnitude/phase matrix [frames √É‚ÄĒ bins]

Algorithm:
  1. Windowing: Apply Hann window to frame
     w[n] = 0.5 * (1 - cos(2√Ź‚ā¨*n / (N-1)))
  2. FFT: FFTJS (CPU) or GPU compute shader (parallel)
     X(k) = √é¬£ x[n]*w[n]*exp(-j*2√Ź‚ā¨*k*n/N)
  3. Extract magnitude & phase:
     Mag[k] = |X(k)|, Phase[k] = arg(X(k))
  4. Power spectrum: P[k] = Mag[k]√ā¬≤ / N√ā¬≤
  5. Log magnitude: dB[k] = 10*log10(P[k] + 1e-10)
  6. Return: {magnitude, phase, power_dB, time_axis}

GPU Optimization:
  √Ę‚Ä†‚Äô Parallel FFT for 4√Ę‚ā¨‚Äú16 frames simultaneously
  √Ę‚Ä†‚Äô Reduces CPU bottleneck by 10√Ę‚ā¨‚Äú100x
```

#### **NODE 4: Voice Activity Detection (VAD)**
```
Purpose: Identify speech-active frames vs. silence/noise
Input:  Power spectrum (dB), noise profile, sample rate
Output: VAD scores [0.0√Ę‚ā¨‚Äú1.0] per frame, binary mask

Hybrid Algorithm:
  1. Energy-based:
     E_frame = mean(P[k]) across all bins
     E_silence = E_noise + 10 dB  (heuristic)
     speech_energy = E_frame > E_silence
     
  2. Spectral flux (derivative):
     √é‚ÄĚ_flux[t] = √é¬£ |dB[t,k] - dB[t-1,k]|
     speech_flux = √é‚ÄĚ_flux > percentile(√é‚ÄĚ_flux, 60)
     
  3. Zero-crossing rate (consonants):
     ZCR[t] = (# sign changes) / frame_length
     speech_zcr = ZCR > 0.1  (tunable)
     
  4. Combine via voting:
     VAD_score[t] = (speech_energy + speech_flux + speech_zcr) / 3
     
  5. Temporal smoothing (coherence):
     VAD_smooth[t] = median(VAD_score[t-2:t+2])
     
  6. Return: scores [0..1], binary mask

Optional: Replace with pre-trained model (Silero VAD, Conformer)
```

#### **NODE 5: Voiceprint Extraction (ECAPA-TDNN)**
```
Purpose: Extract speaker embedding (512-dim) for identity tracking
Input:  Power spectrum (dB), pre-trained model weights
Output: Speaker embedding [512], speaker ID

Architecture:
  Input √Ę‚Ä†‚Äô Conv1D (128 filters) √Ę‚Ä†‚Äô BatchNorm √Ę‚Ä†‚Äô ReLU
  √Ę‚Ä†‚Äô Res2Net (multi-scale blocks) √Ę‚Ä†‚Äô Squeeze-Excitation
  √Ę‚Ä†‚Äô GlobalPooling √Ę‚Ä†‚Äô FC (1024) √Ę‚Ä†‚Äô BatchNorm
  √Ę‚Ä†‚Äô FC (512, embedding) √Ę‚Ä†‚Äô L2-normalize

Process:
  1. Extract VAD-active frames only
  2. Normalize to zero mean, unit variance
  3. Forward pass through model
  4. L2-normalize embedding: e = e / ||e||√Ę‚Äö‚Äö
  5. Compare to existing voiceprints (cosine similarity)
     if sim > 0.9: reuse_existing_speaker
     else: create_new_speaker
  6. Return: {embedding: [512], speaker_id, confidence}

Performance (GPU):
  - 10-sec clip @ 44.1 kHz: ~100ms (Tesla V100)
  - Browser GPU (RTX 3060): ~2√Ę‚ā¨‚Äú3 seconds
  - CPU fallback: ~15√Ę‚ā¨‚Äú30 seconds
```

#### **NODE 6: Multi-Band Noise Reduction**

**6a. Electrical Hum Removal (Notch Filters)**
```
Purpose: Remove 60/120/180/240 Hz (US) or 50/100/150/200 Hz (EU) hum
Input:  Magnitude spectrum, hum frequency, bandwidth
Output: Attenuated spectrum

Algorithm:
  1. Compute FFT bin: target_bin = round(hum_freq * FFT_size / sample_rate)
  2. Define notch width: left_bin, right_bin (typically √ā¬Ī10√Ę‚ā¨‚Äú20 Hz)
  3. Apply notch in spectral domain:
     mag_notched[target_bin] *= 0.01  (-40 dB deep notch)
     for each harmonic_freq in [2*hum, 3*hum, 4*hum]:
       apply_notch(harmonic_freq, magnitude)
  4. Phase preservation: Keep phase unchanged
     X_filtered[k] = mag_notched[k] * exp(j * phase[k])
  5. Return: {magnitude_notched, phase}
```

**6b. Spectral Subtraction (Wiener Filter)**
```
Purpose: Remove steady-state noise across frequencies
Input:  Speech magnitude, noise magnitude, over-subtraction factor (0.5√Ę‚ā¨‚Äú2.0)
Output: Noise-reduced magnitude

Algorithm:
  1. Estimate noise from VAD-inactive frames:
     P_noise[k] = median(P[k] for non-speech frames)
  2. Compute Wiener gain:
     G_wiener[k] = max(0, (P_speech[k] - √é¬Ī*P_noise[k]) / P_speech[k])
     where √é¬Ī = over-subtraction factor (default 1.0)
  3. Apply gain: mag_clean[k] = mag_speech[k] * G_wiener[k]
  4. Clamp to avoid over-subtraction:
     mag_clean[k] = clamp(mag_clean[k], floor=0.01*mag_speech[k])
  5. Phase preservation:
     X_clean[k] = mag_clean[k] * exp(j * phase[k])
  6. Return: {magnitude_clean, phase}

Note: Classic approach; mitigated by Node 8 (harmonic reconstruction)
```

**6c. Adaptive Spectral Gating**
```
Purpose: Frequency-dependent thresholding to preserve speech while suppressing noise
Input:  Power spectrum (dB), noise profile, VAD mask
Output: Gate mask [0.0√Ę‚ā¨‚Äú1.0], dynamic threshold per bin

Algorithm:
  1. Compute SNR per bin:
     SNR_db[k] = P_speech_db[k] - P_noise_db[k]
  2. Set dynamic threshold (frequency-dependent):
     threshold_db[k] = P_noise_db[k] + 6 dB  (6 dB SNR required)
  3. Soft gating via sigmoid:
     gate[k,t] = 1 / (1 + exp(-(SNR_db[k] - threshold_db[k]) / steepness))
     where steepness = 2.0 (controls sharpness)
  4. Temporal smoothing (prevent jitter):
     gate_smooth[k,t] = 0.9*gate_smooth[k,t-1] + 0.1*gate[k,t]
  5. Apply gate:
     mag_gated[k,t] = mag[k,t] * gate_smooth[k,t]
  6. Return: {magnitude_gated, gate_mask}

Tuning:
  - Higher threshold √Ę‚Ä†‚Äô more noise removal (risk of speech loss)
  - Lower threshold √Ę‚Ä†‚Äô preserve speech (residual noise)
  - Sigmoid steepness √Ę‚Ä†‚Äô smooth vs. sharp gating
```

#### **NODE 7: Source Separation (Hybrid Classical + ML)**

**7a. Demucs-Style Hybrid Separation (GPU-Accelerated)**

**7b. Mask-Based Extraction (Voiceprint-Guided)**
```
Purpose: Use speaker embedding to create isolation mask
Input:  Magnitude spectrum, speaker embedding (512-dim), voiceprint library
Output: Speaker mask [0..1], isolated magnitude spectrum

Algorithm:
  1. Extract speaker embedding for entire signal (sliding window)
     embeddings[t] = ECAPA_TDNN(spectrum[t])
  2. Compute similarity to reference voiceprint:
     sim[t,k] = cosine_similarity(embeddings[t], reference_voiceprint)
  3. Convert to mask (soft thresholding):
     mask[k,t] = max(0, sim[t,k] - threshold) / (1 - threshold)
     where threshold = 0.5 (tunable)
  4. Smooth temporally:
     mask_smooth[k,t] = 0.95*mask_smooth[k,t-1] + 0.05*mask[k,t]
  5. Apply mask:
     mag_isolated[k,t] = mag[k,t] * mask_smooth[k,t]
  6. Return: {mask, magnitude_isolated, phase}

```

#### **NODE 8: Harmonic Reconstruction & Formant Enhancement**
```
Purpose: Restore lost harmonic content, boost vowel resonances
Input:  Magnitude spectrum (post-isolation), phase spectrum
Output: Enhanced magnitude spectrum, natural-sounding voice

Algorithm:
  1. Detect formants (vowel resonances) using LPC:
     - Compute Linear Predictive Coefficients (order 12√Ę‚ā¨‚Äú16)
     - Extract poles √Ę‚Ä†‚Äô frequency/bandwidth pairs
     - Typical ranges: F1 (200√Ę‚ā¨‚Äú900 Hz), F2 (500√Ę‚ā¨‚Äú2500 Hz), F3 (1500√Ę‚ā¨‚Äú3500 Hz)
  2. Reconstruct harmonics:
     For each detected formant f:
       magnitude[bin(f)] *= 1.5√Ę‚ā¨‚Äú2.0  (boost, typically +3 to +6 dB)
  3. Smooth spectrum:
     mag_smooth = moving_average(mag, kernel_size=5)
  4. Phase reconstruction (critical for naturalness):
     Use Hilbert transform to ensure min-phase property
  5. Reconstruct time-domain:
     x_time[n] = IFFT(X_enhanced * phase_reconstructed)
  6. Return: {magnitude_enhanced, phase_reconstructed, waveform}
```

#### **NODE 9: Phase Reconstruction (Hilbert Transform)**
```
Purpose: Enforce phase consistency, preserve naturalness
Input:  Magnitude spectrum (post-processing)
Output: Phase spectrum (reconstructed from magnitude)

Algorithm:
  1. Compute minimum-phase impulse response via cepstral method:
     - Take log of magnitude: log_mag = log(|X(f)|)
     - Apply IFFT: c[n] = IFFT(log_mag)  (cepstrum)
     - Zero out negative lags: c_min[n] = 2*c[n] for n>0, else 0
     - Apply FFT: log_H = FFT(c_min)
     - Exponentiate: H = exp(log_H)
  2. Extract phase: phase_min = arg(H)
  3. Return: {phase_min, H}

Justification:
  - Minimum-phase is natural for speech (causal)
  - Preserves perceptual quality while removing phase noise
  - Eliminates musical artifacts from spectral subtraction
```

#### **NODE 10: Anti-Aliasing & Dereverberation**
```
Purpose: Smooth spectral edges, remove room reflections
Input:  Magnitude spectrum, phase, estimated RIR
Output: Dereverberated waveform

Algorithm:
  1. Spectral smoothing (anti-aliasing):
     mag_smooth[k] = (mag[k-1] + 2*mag[k] + mag[k+1]) / 4
  2. Dereverberation (if RIR available):
     H_room = FFT(RIR)
     X_anechoic √Ę‚ÄįňÜ X_observed / H_room
  3. Dereverberation (if RIR unavailable):
     √Ę‚Ä†‚Äô Assume exponential decay: X_anechoic[k] = X_observed[k] * (1 - decay_factor)
     √Ę‚Ä†‚Äô decay_factor √ĘňÜňÜ [0.1, 0.5] (tunable)
  4. Combine results:
     X_processed = (X_dereverberated + X_smooth) / 2
  5. Reconstruct time-domain:
     x_final = IFFT(X_processed * phase)
  6. Return: {waveform_dereverberated, impulse_response_estimate}
```

#### **NODE 11: Post-Processing & Normalization**
```
Purpose: Final audio quality assurance, loudness alignment
Input:  Processed waveform, original duration, loudness target (LUFS)
Output: Final normalized waveform ready for export

Algorithm:
  1. Clip protection:
     x_clipped[n] = clamp(x[n], -0.99, +0.99)
  2. Fade edges (prevent clicks):
     Apply 50ms linear fade-in at start
     Apply 50ms linear fade-out at end
  3. Loudness normalization (ITU-R BS.1770-4):
     - Compute LUFS (loudness units relative to full scale)
     - Target LUFS: Podcast = -16, Music = -14
     - Compute gain: gain_dB = target_LUFS - current_LUFS
     - Apply: x_normalized[n] = x[n] * 10^(gain_dB/20)
  4. True peak limiting (prevent inter-sample peaks):
     Use cubic interpolation to detect peaks between samples
     if peak > 0.0 dBFS: apply soft clipper
  5. Final RMS level check:
     RMS = sqrt(mean(x√ā¬≤))
     if RMS < -50 dBFS: warn "very quiet"
     if RMS > -3 dBFS: warn "very loud, clipping risk"
  6. Return: {waveform_final, metadata: {LUFS, RMS, peak_dBFS, duration}}
```

### 7.1 Low-Latency Strategies (Real-Time Mode)
```
Challenge: 12 DSP nodes √É‚ÄĒ ~12ms latency = 144ms round-trip (unacceptable)

Solutions:

   - Use 2048-point FFT (~45ms @ 44.1 kHz) instead of 4096
   - Trade-off: Lower frequency resolution (21 Hz/bin vs 10 Hz/bin)
   - Sufficient for speech (15 bins/formant)

2. Overlap-Add with 50% Window
   - Process every 1024 samples
   - Enables streaming with minimal buffering
   - Example: 1024 samples @ 44.1 kHz = 23.2ms

3. Parallel Processing (Web Workers)

   - Reduces CPU load by 10√Ę‚ā¨‚Äú100x
```

### 7.2 Memory Optimization
```
Challenge: 2-hour podcast @ 44.1 kHz stereo = 633 MB (unacceptable)

Solutions:

1. Chunked Processing
   - Process 1-minute chunks instead of entire file
   - Memory footprint: ~2 MB per chunk + DSP buffers (~5 MB total)

2. Efficient Data Structures
   - Use Float32Array (4 bytes/sample) not regular arrays (8 bytes + overhead)
   - Preallocate buffers, reuse across frames

3. Web Worker Memory

   - Requires HTTPS + cross-origin isolation headers
   - Browser support: Chrome, Firefox, Edge (Safari pending)
   - Saves 7 MB per additional worker (8 workers = 56 MB saved)

4. Garbage Collection Pauses
   - Avoid temporary allocations in hot loops
   - Pre-allocate output buffers
   - Reuse buffers via .set() instead of slice()

5. Compression & Caching
   - Store processed audio as MP3 locally (10% of WAV size)
   - Cache model weights after first download
```
